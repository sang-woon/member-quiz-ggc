# PRD (제품 요구사항 정의서)

> 의원얼굴퀴즈: 직원을 위한 의원 얼굴 학습 툴

---

## MVP 캡슐

| # | 항목 | 내용 |
|---|------|------|
| 1 | 목표 | 경기도의회 직원이 의원 얼굴과 이름을 쉽고 재미있게 학습 |
| 2 | 페르소나 | 경기도의회 사무처 직원 (특히 신규 입사자) |
| 3 | 핵심 기능 | FEAT-1: 얼굴 보고 이름 맞추기 퀴즈 |
| 4 | 성공 지표 (노스스타) | 사용자가 전체 의원 정답률 80% 이상 달성 |
| 5 | 입력 지표 | 퀴즈 완료 횟수, 일일 활성 사용자 |
| 6 | 비기능 요구 | 모바일/PC 반응형 지원 |

| 10 | 다음 단계 | 의원 데이터 수집 및 정리 |

---

## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

경기도의회 사무처 직원들은 의원님들의 얼굴을 익히기 어려워합니다.
마치 새 학교에 전학 온 학생이 150명 넘는 반 친구들 얼굴을 모두 외워야 하는 것과 같습니다.
의원님을 직접 만났을 때 누구인지 몰라 당황하는 상황이 종종 발생합니다.

### 1.2 목표 상황 (To-Be)

직원들이 게임처럼 재미있게 의원님들의 얼굴과 이름, 소속 지역구/위원회를 학습하여
업무 중 의원님을 만났을 때 자신 있게 응대할 수 있습니다.

### 1.3 왜 지금인가? (Why Now)

- 11대 경기도의회가 새롭게 구성되어 의원 구성이 변경됨
- 신규 입사 직원들의 적응을 돕기 위한 학습 도구 필요
- 기존에는 효과적인 학습 도구가 없었음

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "열심히 배우는 신입 김주임" |
| 나이/직업 | 28세 경기도의회 사무처 신규 직원 |
| 핵심 니즈 | 빠르게 의원님들 얼굴 익히기 |
| 불편 포인트 | 150명 넘는 의원님 얼굴을 단순 암기로 외우기 어려움 |
| 사용 상황 | 출퇴근 시간, 점심시간, 자투리 시간에 자유롭게 학습 |

### 2.2 페르소나의 하루 (시나리오)

신입 김주임은 출근 후 업무 시작 전 5분간 의원얼굴퀴즈를 열어
오늘의 퀴즈 10문제를 풉니다. 어제 틀렸던 의원님 3명이 다시 나왔고,
이번에는 모두 맞췄습니다. 점심시간에도 짧게 5분간 퀴즈를 풀며
지역구별로 필터링해서 자신이 담당하는 지역 의원님들을 집중 학습합니다.

## 3. 사용자 스토리

### FEAT-0: 진입 (온보딩/로그인 없음)
- [x] 사용자로서, 로그인 없이 바로 퀴즈를 시작할 수 있어야 한다. 왜냐하면 진입 장벽을 낮춰 언제든 학습 가능해야 하기 때문.

### FEAT-1: 얼굴 보고 이름 맞추기 퀴즈 (MVP)
- [ ] 직원으로서, 의원 사진을 보고 4개 보기 중 정답을 고를 수 있어야 한다. 왜냐하면 객관식이 부담 없이 학습하기 좋기 때문.
- [ ] 직원으로서, 정답/오답 여부를 즉시 확인할 수 있어야 한다. 왜냐하면 즉각적인 피드백이 학습 효과를 높이기 때문.
- [ ] 직원으로서, 오답 시 정답을 보여주고 해당 의원 정보를 확인할 수 있어야 한다. 왜냐하면 틀린 것을 바로 교정해야 기억에 남기 때문.
- [ ] 직원으로서, 지역구나 소속 위원회로 필터링하여 퀴즈를 풀 수 있어야 한다. 왜냐하면 담당 업무와 관련된 의원님을 집중 학습할 수 있기 때문.

### FEAT-2: 이름 보고 얼굴 맞추기 (v2 이후)
- [ ] 직원으로서, 의원 이름을 보고 4개 사진 중 정답을 고를 수 있어야 한다.

### FEAT-3: 학습 진도 저장 (v2 이후)
- [ ] 직원으로서, 내 정답률과 학습 이력을 확인할 수 있어야 한다.

## 4. 성공 지표

### 4.1 노스스타 지표 (결과 지표)
| 지표명 | 정의 | 목표치 | 측정 주기 |
|--------|------|--------|----------|
| 전체 의원 정답률 80%+ 달성자 | 전체 의원 퀴즈에서 80% 이상 맞춘 사용자 | MVP에서는 측정 안 함 | - |

### 4.2 입력 지표 (행동 지표)
| 지표명 | 정의 | 목표치 |
|--------|------|--------|
| 퀴즈 완료 수 | 사용자가 완료한 총 퀴즈 세트 수 | MVP에서는 측정 안 함 |
| 재방문율 | 다시 앱에 접속한 비율 | MVP에서는 측정 안 함 |

*참고: MVP에서는 별도 지표 측정 없이 순수 기능 구현에 집중*

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- FEAT-1: 얼굴 보고 이름 맞추기 퀴즈
- FEAT-1: 지역구/위원회 필터링
- 로그인 없이 바로 사용

### 5.2 옵션 (시간 허용 시)
- 정답/오답 애니메이션 효과 (듀오링고 스타일)
- 퀴즈 난이도 조절 (쉬움/보통/어려움)

### 5.3 Non-goals (명시적 제외)
- 로그인/회원가입 기능 - 진입 장벽 낮추기 위해 제외
- 학습 진도 저장 - MVP에서는 매번 새로 시작
- 랭킹/리더보드 - v2에서 검토
- 다른 지방의회 확장 - 경기도의회 우선

## 7. 해결 가설과 실험

### 7.1 가설
> 만약 의원 얼굴 퀴즈를 게임처럼 재미있게 제공하면,
> 직원들이 자발적으로 학습하여 의원 인식률이 높아질 것이다.
> 왜냐하면 단순 암기보다 게임화된 반복 학습이 더 효과적이기 때문이다.

### 7.2 실험 루프

```
실험 → 관측 → 학습 → 다음 가설
```

| 단계 | 내용 |
|------|------|
| 실험 | MVP 출시 후 동료들에게 사용 요청 |
| 관측 | 실제 사용 여부, 재방문 여부 |
| 학습 | 어떤 기능이 도움되는지 피드백 수집 |
| 다음 | 학습 진도 저장, 오답 노트 기능 검토 |
